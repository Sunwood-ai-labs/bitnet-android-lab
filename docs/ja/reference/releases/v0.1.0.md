# v0.1.0 リリースノート

![v0.1.0 release header](/releases/release-header-v0.1.0.svg)

`v0.1.0` は `bitnet-android-lab` の初回公開リリースです。この repository には前の tag がないため、このノートは root commit から 2026-03-25 時点の `main` までに含まれる公開履歴全体を対象にしています。

## ハイライト

- QVAC Fabric BitNet LoRA を実機 Android 端末で試した結果を、Termux、`adb forward`、patched local Vulkan build の経路で再確認しやすい形にまとめました。
- Termux 側の fine-tuning / inference helper script と、Android 固有 workaround を切り出した最小 patch を公開し、fork した upstream tree を丸ごと同梱しない形にしました。
- Sanitized evidence、日付付き rerun snippet、端末仕様、throughput 参照値を公開し、public claim を実行結果に結び付けています。

## 監視とツール類

- [`scripts/termux/install_monitoring_tools.sh`](https://github.com/Sunwood-ai-labs/bitnet-android-lab/blob/main/scripts/termux/install_monitoring_tools.sh) で `gotop`、`htop`、`bmon` を任意導入できます。
- [`scripts/windows/watch_android_resources.ps1`](https://github.com/Sunwood-ai-labs/bitnet-android-lab/blob/main/scripts/windows/watch_android_resources.ps1) は、Windows 側から `adb` を通してメモリ / スワップのバー、各コア使用率バー、周波数、`top`、`dumpsys cpuinfo`、`dumpsys meminfo` を 1 画面で観察できます。
- docs では、これらを benchmark 計測器ではなく live observation helper として明示しています。

## Docs と公開面

- 英語 / 日本語の VitePress docs site、公開アセット、GitHub Pages deployment を追加しました。
- CI では docs build、shell script syntax、PowerShell parse を確認してから Pages workflow が公開します。
- Windows から Termux へ入る bootstrap 経路については、再利用可能な [`android-termux-ssh-bootstrap`](https://github.com/Sunwood-ai-labs/android-termux-ssh-bootstrap-skill) skill を使ったことも setup guide に記録しました。

## Evidence の範囲

- [`evidence/manifest.md`](https://github.com/Sunwood-ai-labs/bitnet-android-lab/blob/main/evidence/manifest.md) が public claim と script / log snippet を対応付けています。
- tiny training の checkpoint progression、fast checkpoint inference、2026-03-25 の TQ1 / TQ2 rerun throughput snippet を公開対象に含めました。
- この release の主張は狭く保っています。実機 1 台、patched local Termux path、`FORTIFY`、中間 checkpoint artifact、benchmark ではない `tok/s` という caveat を明記しています。
